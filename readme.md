### на разработку LoRA-адаптера для модели Qwen3.5-9B в домене «Онкология ЖКТ»

**Редакция:** 2.0

## 1. Общие сведения

**1.1. Полное наименование работы:** Разработка и обучение LoRA-адаптера для специализированной системы поддержки принятия врачебных решений (СППВР) в области онкологии желудочно-кишечного тракта на базе модели Qwen3.5-9B с использованием автоматизированной верификации QA-датасета гетерогенным LLM-ансамблем.

**1.2. Базовая модель:** Qwen3.5-9B (Qwen/Qwen3.5-9B).

**1.3. Платформа работы по созданию адаптеров:** Together.ai (inference + fine-tuning). Ключи доступа сделаю.

**1.4. Область применения:** Клиническая онкология ЖКТ — диагностика, стадирование, терапия, хирургическое лечение, химиотерапия, лучевая терапия, нутритивная поддержка, паллиативная помощь.

**1.5. Ключевое отличие от редакции 1.0:** Экспертная верификация QA-датасета осуществляется гетерогенным ансамблем из 4-6 LLM-моделей с выборочной калибровкой на подвыборке, оцененной врачами-онкологами.

## 5. Требования к техническому решению

### 5.1. Базовая модель

| Параметр | Значение |
|---|---|
| Модель | Qwen3.5-9B |
| Архитектура | Dense + Hybrid (Gated Delta Networks) |
| Параметры | 9 млрд |
| Контекстное окно | 262 144 токена |
| Языки | 201 язык, включая русский |
| Лицензия | Apache 2.0 |

### 5.2. Параметры LoRA-адаптера (полное покрытие)

| Параметр | Значение | Обоснование |
|---|---|---|
| Ранг (r) | 64 | Достаточная ёмкость при большом датасете |
| Alpha | 128 | Соотношение alpha/r = 2 |
| Target modules | `q_proj`, `k_proj`, `v_proj`, `o_proj`, `gate_proj`, `up_proj`, `down_proj` | Полное покрытие attention + MLP для усвоения фактологических знаний |
| Dropout | 0.05 | Защита от переобучения |
| Квантование | 4-bit NF4 | QLoRA для экономии памяти |
| Тип данных | bfloat16 | Для стабильности обучения |
| Метод | LoRA+ | Разные learning rate для A и B, ускорение сходимости |

LoRA+ изучить и взять из работы - Hayou et al., 2024 («LoRA+: Efficient Low Rank Adaptation of Large Models»)


**Обоснование полного покрытия MLP:** при датасете >8000 QA-пар полное покрытие MLP оправдано, так как MLP-слои ответственны за хранение фактических знаний (дозировки, протоколы, показания), что критично для медицинского домена.

### 5.3. Параметры обучения

| Параметр | Значение                                                                       |
|---|--------------------------------------------------------------------------------|
| Оптимизатор | AdamW (β₁=0.9, β₂=0.999)                                                       |
| Learning rate | 2×10⁻⁴ с cosine-расписанием                                                    |
| Warmup | 5% шагов                                                                       |
| Batch size | 8 (gradient accumulation = 4)                                                  |
| Эпохи | 3–5 (с ранней остановкой по validation loss)                                   |
| Max seq length | устанавливается после подсчета максимального количества токенов в парах QA+10% |
| Weight decay | 0.01                                                                           |
| Max grad norm | 1.0                                                                            |

### 5.4. Конфигурация датасета и верификации


### Этап 3. Верификация QA-датасета LLM-ансамблем 

**3.1. Многоуровневая фильтрация:**

```
[Сырые QA-пары: 15 000]
        │
        ▼
[Уровень 1: Автоматическая проверка]
  - Groundedness (RAG-поиск в источниках, порог ≥ 0.8)
  - Completeness (покрытие сущностей ≥ 0.9)
  - Hallucination detection (NLI-модель DeBERTa-v3)
        │
        ▼
[Отсев 20%: 12 000 QA]
        │
        ▼
[Уровень 2: LLM-ансамбль (4-6 моделей)]
  - Параллельный запрос ко всем моделям
  - Расчёт S_final, Consensus, Confidence, Disagreement
        │
        ▼
[Фильтрация: S_final ≥ 7.0, C ≥ 0.6, Conf ≥ 0.67]
  - Отсев 15%: 10 200 QA
        │
        ▼
[Уровень 3: Обработка расхождений]
  - Если max(sᵢ) - min(sᵢ) > 4: tiebreaker (уточним каким open-source с расширенным промптом)
  - Если расхождение сохраняется → отклонить
        │
        ▼
[Уровень 4: Калибровка ансамбля]
  - 200 случайных QA оцениваются врачами (2-3 онколога)
  - Расчёт ρ = Pearson(S_final, S_expert)
  - Критерий: ρ ≥ 0.75
        │
        ▼
[Уровень 5: Финальная валидация]
  - 5% случайных QA (500 пар) оцениваются врачами - под вопросом, уточним позже
  - Подтверждение калибровки
        │
        ▼
[Финальный датасет: ~10 000 QA-пар]
```

**3.2. Математическая модель агрегации:**

$$S_{final}(q, a) = \frac{\sum_{i=1}^{6} w_i \cdot s_i}{\sum_{i=1}^{6} w_i}$$

$$C(q, a) = 1 - \frac{\sigma(s_1, \ldots, s_6)}{\max(s_i) - \min(s_i) + \epsilon}$$

$$\text{Conf}(q, a) = \frac{1}{6} \sum_{i=1}^{6} \mathbb{1}[s_i \geq 7]$$

**3.3. Критерий принятия QA-пары:**

$$\text{Accept}(q, a) \iff \begin{cases}
S_{final} \geq 7.0 \\
C(q, a) \geq 0.6 \\
\text{Conf}(q, a) \geq 0.67 \\
D(q, a) = 0 \text{ ИЛИ } \text{tiebreaker} \geq 7.0
\end{cases}$$

**3.4. Калибровочный протокол:**
1. Случайная выборка 200 QA-пар;
2. Оценка LLM-ансамблем → $S_{final}$;
3. Оценка 3 врачами-онкологами → $S_{expert}$;
4. Расчёт $\rho = \text{Pearson}(S_{final}, S_{expert})$;
5. При $\rho < 0.75$ — итеративная настройка весов $w_i$;
6. Построение калибровочной кривой $P(\text{correct} | S_{final})$.

**Результат:** `dataset_verified.jsonl`, отчёт о калибровке, калибровочная кривая.

### Этап 5. Тестирование и оценка качества

**5.1. Автоматические метрики:**

| Метрика | Формула / описание | Порог |
|---|---|---|
| **Clinical Accuracy** | Доля клинически корректных ответов на test-set | ≥ 0.85 |
| **Safety Score** | Доля безопасных ответов на adversarial-вопросах | ≥ 0.95 |
| **Refusal Rate** | Доля корректных отказов при отсутствии данных в КР | ≥ 0.90 |
| **Guideline Alignment** | Соответствие ответов формализованным КР | ≥ 0.80 |
| **ECE** | Expected Calibration Error | ≤ 0.10 |
| **Domain Coverage** | Покрытие сущностей онтологии | ≥ 0.90 |

**5.2. Тестовые сценарии:**

**Тест 1. Фактологическая точность (200 вопросов):**
- Вопросы на конкретные факты из КР;
- Оценка: точное совпадение / семантическая эквивалентность.

**Тест 2. Клинические сценарии (100 кейсов):**
- Реальные клинические случаи с коморбидностью;
- Оценка: LLM-ансамблем по 10-балльной шкале.

**Тест 3. Безопасность и отказы (100 вопросов):**
- Вопросы вне области КР, с устаревшими данными, провокационные;
- Оценка: корректность отказа или перенаправления.

**Тест 4. Противоречивость (50 парных вопросов):**
- Один и тот же вопрос в разной формулировке;
- Оценка: консистентность ответов.

**Тест 5. A/B-тестирование с базовой моделью:**
- 200 вопросов к Qwen3.5-9B base vs Qwen3.5-9B + LoRA;
- Оценка: LLM-ансамблем (Elo-рейтинг).

**Тест 6. Валидация LLM-ансамбля:**
- 200 QA-пар оцениваются LLM-ансамблем и 3 врачами;
- Расчёт Pearson correlation $\rho$;
- Построение калибровочной кривой;
- Критерий: $\rho \geq 0.75$.

**5.3. Выборочная экспертная оценка:**
- 5% тестовых ответов (50 пар) оцениваются врачами-онкологами;
- Расчёт Cohen's κ между LLM-ансамблем и экспертами;
- Критерий: κ ≥ 0.70.

**Результат:** отчёт о тестировании, примеры ответов, рекомендации.

## 9. Риски и их разрешение

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Систематическая ошибка LLM-ансамбля | Средняя | Высокое | Гетерогенный ансамбль + калибровка |
| Низкая корреляция ансамбля с экспертами | Низкая | Высокое | Итеративная настройка весов $w_i$ |
| Противоречия между КР | Высокая | Высокое | Темпорально-осознанное разрешение (Теорема 2) |
| Галлюцинации модели | Средняя | Критическое | Safety-aware loss + Refusal Training |
| Расхождения в оценках ансамбля | Высокая | Среднее | Tiebreaker модель + отклонение |
| Переобучение | Средняя | Среднее | Ранняя остановка, dropout, валидация |
| Устаревание КР | Низкая | Высокое | Версионирование, механизм дообучения |
| Задержки API | Средняя | Низкое | Асинхронные запросы + retry |

## 10. Научная новизна

### Позиция 1 (Формальная модель клинических рекомендаций)
**Впервые предложена** формальная модель представления КР в виде кортежа $\langle \text{Cond}, \text{Act}, \text{Ctx}, \text{LoE}, \text{SoR}, \text{Ver} \rangle$ с терминологической привязкой к МКБ-11, SNOMED CT, LOINC.

### Позиция 2 (Метод обеспечения покрытия клинических сценариев)
**Разработан** метод coverage-aware генерации клинических вопросов на основе извлечения медицинских сущностей и стратификации по типам запросов. **Доказана теорема** о нижней границе покрытия пространства клинических сценариев.

### Позиция 3 (Алгоритм темпорально-осознанного разрешения конфликтов)
**Предложен** алгоритм разрешения семантических конфликтов между версиями КР на основе приоритета даты актуализации и уровня доказательности. **Доказана теорема** о непротиворечивости результирующей базы знаний.

### Позиция 4 (Надёжность LLM-ансамбля как верификатора) — НОВАЯ
**Впервые предложен и теоретически обоснован** метод верификации медицинских QA-датасетов посредством гетерогенного LLM-ансамбля с оптимальным взвешиванием. **Доказана теорема** о надёжности ансамбля:

$$\epsilon_{ensemble}^2 = \frac{1}{\sum_{i=1}^{k} 1/\sigma_i^2} < \min_i \sigma_i^2$$

$$P(\text{false positive}) \leq \exp\left(-\frac{k \cdot \Delta^2}{8 \sigma_{max}^2}\right)$$

При $k = 6$ и $\sigma_i \approx 1.5$ обеспечивается надёжность >95% при снижении стоимости верификации на 96% по сравнению с традиционным подходом.

### Позиция 5 (Гарантии клинической корректности СППВР)

Впервые получена теоретическая нижняя граница клинической корректности СППВР:

$$
\text{Acc}_{\text{clin}}(M_\theta) \geq \tau_{\text{cov}} \cdot (1 - \epsilon_{KL})
$$
